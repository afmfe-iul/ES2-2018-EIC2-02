As únicas funcionalidades que o grupo optou por não implementar foram a possibilidade de escolher algoritmos de forma mista, pois a funcionalidade manual dá a mesma liberdade ao utilizador que a forma mista daria. A segunda funcionalidade que o grupo optou por não implementar foi a compilação dos documentos R e .tex para pdf e eps, pois para fazer isto estariamos a criar dependências da nossa aplicação com outras (miktex e cran.r-project.org), o que obrigaria os utilizadores do nosso jar a ter essas aplicações nos seus computadores. Assim o que é esperado é que cada utilizador utilize a sua aplicação favorita para visualização de ficheiros R e .tex, que estarão acessíveis na pasta que este definiu para output.
